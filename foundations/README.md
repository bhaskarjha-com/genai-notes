# Foundations

> Transformer architectures, embeddings, tokenization, and representation fundamentals.

## Notes in This Directory

| Note | Difficulty | Description |
|------|:----------:|-------------|
| [Attention Mechanism Deep Dive](attention-deep-dive.md) | Expert | A complete treatment of attention variants, their memory/compute tradeoffs, KV-cache mechanics, and hardware-efficient implementations (FlashAttention) |
| [Attention Mechanism](attention-mechanism.md) | Intermediate | A mechanism that lets each element in a sequence dynamically focus on relevant parts of the entire input |
| [Embeddings](embeddings.md) | Intermediate | Dense vector representations that encode the meaning/semantics of data (text, images, audio) into fixed-size arrays of numbers |
| [Modern LLM Architectures](modern-architectures.md) | Advanced | The architectural innovations that make modern LLMs (GPT-5, LLaMA 4, Gemini 3) work - beyond the basic Transformer |
| [Scaling Laws & Pre-training](scaling-laws-and-pretraining.md) | Advanced | The process of training an LLM from scratch on internet-scale data, and the mathematical laws predicting how performance improves with more compute, data, and parameters |
| [State Space Models](state-space-models.md) | Expert | A family of sequence models based on continuous state-space representations that process sequences in linear time, offering an alternative to the quadratic attention mechanism in transformers |
| [Tokenization](tokenization.md) | Intermediate | The process of breaking text into sub-word units (tokens) that LLMs actually process |
| [Transformers](transformers.md) | Intermediate | A neural network architecture based on self-attention that processes entire sequences in parallel |
